# 评测体系与上线门槛

## 1. 为什么不能只看模型回答

本项目评估的是完整产品链路，而不是单个 JSON：

```text
文件进入
  -> 解析
  -> 结构生成
  -> 校验
  -> 预览与编辑
  -> 用户确认
  -> 创建项目
```

因此评测同时覆盖模型端、工程端和用户体验。

## 2. 100 分评测框架

| 维度 | 权重 | 重点 |
| --- | ---: | --- |
| 输出可用性与 schema 合规 | 15 | 是否成功解析、字段是否合法、输出是否完整 |
| 项目与表格理解 | 15 | 主数据区、项目名称、业务语义是否正确 |
| 结构还原质量 | 25 | 模块/Card/任务/子任务路径、数量和顺序 |
| 任务字段抽取质量 | 25 | 标题、负责人、时间、备注和标签 |
| 稳定性与性能 | 10 | 重复运行一致性、耗时、截断与失败 |
| 可审查性与产品可用性 | 10 | 预览清晰度、问题定位和人工修正成本 |
| 合计 | 100 |  |

## 3. AI + 人工联合评分

### 自动评测负责

- JSON 与 schema 合规。
- 节点数量和覆盖率。
- 完整路径匹配。
- 有值字段命中率。
- 日期格式与逻辑冲突。
- 重复运行 diff。
- 耗时、重试和失败原因。

### 人工复核负责

- 项目和分组语义是否合理。
- 父子关系是否符合真实业务。
- 任务名称是否可执行。
- 预览是否容易理解和修改。
- 错误是否影响项目可用性。
- 缺陷严重度与最终发布建议。

自动评测是辅助判断，不取代人工业务验收。

## 4. 关键指标

### 4.1 结构指标

```text
节点覆盖率 = 正确映射节点数 / Gold Standard 节点数
完整路径准确率 = 正确“项目/模块/Card/任务”路径数 / 路径总数
字段准确率 = 正确字段数 / Gold Standard 中有值字段数
```

只对任务标题做匹配会遗漏“任务名称正确但分组错误”的问题，因此必须引入完整路径。

### 4.2 稳定性指标

同一文件至少重复运行多次，比较：

- 节点增删 diff。
- 层级路径 diff。
- 关键字段 diff。
- warnings diff。
- 失败率与重试次数。

### 4.3 性能指标

- 端到端平均耗时。
- P50 / P95 耗时。
- 解析、模型、校验和创建各阶段耗时。
- 超时率与后台恢复成功率。

平均耗时不能掩盖长尾。对于“极速”体验，P95 和恢复能力比单次最快结果更有意义。

## 5. 样例分层

| 层级 | 典型特征 | 评测重点 |
| --- | --- | --- |
| Tier 1 | 单表、扁平任务清单 | 基础字段与创建成功率 |
| Tier 2 | 多级项目计划、常见合并单元格 | 分组、路径和日期字段 |
| Tier 3 | 多 sheet、复杂工程计划、非标准布局 | 主数据区识别、语义和降级 |

支持范围和通过率需要按复杂度分别报告，避免简单样例掩盖复杂样例失败。

## 6. 脱敏版本快照

v0.3 回归结果：

| 指标 | 结果 |
| --- | ---: |
| 样例数 | 20 |
| AI 自动评测平均分 | 89.5 |
| 人工复核平均分 | 83.1 |
| AI 判定 pass / warning / fail | 18 / 2 / 0 |
| 人工判定通过 / 有条件通过 / 不通过 | 13 / 2 / 5 |
| 平均耗时 | 40.1 秒 |

自动结果与人工结论的差异说明：

- schema 合规不代表分组合理。
- 节点数量一致不代表父子路径正确。
- 高平均分不能覆盖少数严重失败。
- 业务可用性需要人工或更强的语义评测补足。

## 7. 建议上线门槛

以下是作品集中采用的决策框架，实际阈值应随业务风险调整：

- 不存在阻断创建、严重漏任务、越权或重复创建缺陷。
- Tier 1 / Tier 2 主支持范围达到既定人工通过率。
- 完整路径准确率和有值字段准确率达到版本目标。
- 同文档重复运行 diff 在可接受范围。
- P95 耗时、超时率和后台恢复达到体验目标。
- 所有高风险 warning 在预览或人工确认环节有明确处理。
- 自动评测和人工复核分歧样例完成归因。

## 8. 缺陷分级

| 等级 | 示例 | 发布影响 |
| --- | --- | --- |
| P0 阻断 | 无法创建、数据越权、重复写入严重 | 必须阻断发布 |
| P1 严重 | 大量任务丢失、核心层级错误、结果不可用 | 修复并回归后发布 |
| P2 一般 | 局部字段错误、需要明显人工修改 | 评估范围与灰度 |
| P3 轻微 | 文案、排序或低影响展示问题 | 可进入后续迭代 |

发布结论同时看平均指标和严重缺陷；任何 P0 都不能被高平均分抵消。
