# 公开版 PRD：Excel 一键生成项目计划

> 文档性质：脱敏作品集材料。品牌、客户、接口和商业规则均已抽象，不代表生产系统的完整实现。

## 1. 背景与目标

业务团队经常用 Excel 维护项目排期、活动计划和任务清单。当计划需要进入项目管理系统协作时，用户要重新创建项目、分组和任务，既耗时又容易漏项。

本功能希望把这条链路缩短为：

```text
选择 Excel
  → 解析结构
  → 生成项目预览
  → 用户校对
  → 确认创建
```

产品目标不是让 AI 直接替用户做不可逆写入，而是先提供一个可理解、可编辑、可追溯的项目草稿。

### 成功标准

- 标准任务表可以稳定生成“项目 → 模块 → 卡片 → 任务”层级。
- 每个生成节点可以回溯到源工作表和行号。
- 生成过程可最小化到后台，完成后可以恢复。
- 只有用户确认后才创建正式项目。
- 缺失字段、日期冲突和默认分组必须显式提示。

## 2. 用户与使用场景

### 目标用户

- 已经用 Excel 管理工作、但希望迁移到协作工具的项目负责人。
- 接收外部计划表并需要快速拆解任务的运营、产品和交付人员。
- 需要先验证 AI 结果、再决定是否创建项目的谨慎型用户。

### 典型场景

1. 在表格编辑页点击“一键转计划”，直接读取当前工作簿。
2. 从新建页导入 Excel，生成结构化项目预览。
3. 生成耗时较长时最小化窗口，稍后从任务气泡恢复。
4. 在预览中发现结构或日期问题，修改后再确认创建。

## 3. 范围

### 本版本包含

- `.xlsx` 文件输入。
- 表头识别和基础字段映射。
- 项目、模块、卡片、任务四级结构。
- 生成进度、后台恢复和历史任务入口。
- 看板式预览、质量提示和确认创建。
- 重复提交保护与结果状态反馈。

### 本版本不包含

- 图片、颜色、批注等视觉语义的稳定识别。
- 对任意复杂甘特图、跨表公式和合并单元格的完整支持。
- 未经用户确认直接写入正式项目。
- 在公开仓库中提供生产接口、鉴权或真实客户数据。

## 4. 核心流程

### 4.1 主流程

1. 用户从表格页或导入页发起生成。
2. 系统校验文件类型、大小和可读取性。
3. 后台任务依次执行：读取工作表、识别表头、抽取行事实、生成层级、规则校验。
4. 前端展示阶段进度；用户可最小化，不中断后台任务。
5. 生成完成后展示项目摘要、看板预览、警告和来源信息。
6. 用户修改或确认结果。
7. 系统通过幂等键创建项目并返回结果页。

### 4.2 状态机

```text
READY
  → PARSING
  → STRUCTURING
  → VALIDATING
  → PREVIEW_READY
  → CREATING
  → CREATED

任一生成阶段 → FAILED / CANCELLED
CREATING → CREATED / CREATE_FAILED
```

生成任务和创建任务分开建模，避免“预览成功但写入失败”时无法恢复，也避免重复点击造成重复项目。

## 5. 页面与交互

### 5.1 生成弹窗

- 展示来源、文件名、工作表数量和能力边界。
- 主按钮为“生成项目预览”。
- 对复杂表格给出风险提示，不用模糊的“AI 可能出错”代替可执行说明。

### 5.2 生成进度

- 展示当前阶段，不伪造精确到秒的完成时间。
- 支持“最小化到后台”。
- 关闭弹窗时，如果任务仍在运行，默认转入后台而不是终止。

### 5.3 后台任务气泡

- 运行中展示阶段和进度。
- 完成后切换为“查看预览”。
- 失败时展示简明原因及重试入口。

### 5.4 看板预览

- 顶部展示项目名、来源、耗时及节点数量。
- 主区域展示模块、卡片与任务。
- 警告按严重度区分：阻断、需确认、提示。
- 节点支持查看 `source_ref`，定位到原始工作表行。

### 5.5 确认创建

- 只有阻断级问题全部处理后按钮才可用。
- 创建操作使用幂等键；重复点击返回同一任务状态。
- 创建完成后可以跳转项目，也可以继续处理下一个文件。

## 6. 数据层级

| 层级 | 说明 | 典型来源 |
| --- | --- | --- |
| Project | 整份计划 | 文件名、标题区 |
| Module | 一级阶段或业务分组 | 模块/阶段列 |
| Card | 可交付工作包 | 卡片/里程碑/工作包列 |
| Task | 可执行任务 | 任务名称列 |

任务可包含负责人、开始日期、截止日期、备注和来源定位。缺失模块或卡片时允许使用默认分组，但必须生成 warning。

## 7. 异常与降级

| 场景 | 产品处理 |
| --- | --- |
| 文件不可读取 | 阻断，说明支持格式并允许重新选择 |
| 无法识别任务列 | 阻断，建议用户选择表头行 |
| 模块/卡片缺失 | 使用默认分组并提示 |
| 日期格式异常 | 保留原始值，不静默猜测 |
| 开始日期晚于截止日期 | 标记需确认 |
| 表格超出稳定能力范围 | 允许预览，但不承诺一键创建 |
| 创建接口超时 | 保留生成结果，允许安全重试 |

## 8. 埋点与诊断

至少记录以下事件及统一 `task_id`：

- `generate_started`
- `parse_succeeded` / `parse_failed`
- `preview_ready`
- `preview_edited`
- `create_confirmed`
- `create_succeeded` / `create_failed`
- `background_minimized` / `background_restored`

诊断日志记录阶段耗时、输入复杂度、warning 类型和失败原因，不记录原始文档正文。

## 9. 验收标准

- 示例 Excel 能生成预期层级，任务覆盖率为 100%。
- 同一输入重复运行时，确定性字段完全一致。
- 所有任务带有工作表和行号来源。
- 空任务名不会进入结果，并生成明确 warning。
- 日期冲突能被检测。
- 后台任务在弹窗关闭后继续运行，并可恢复预览。
- 重复确认不会创建重复项目。
- 公开 Demo、测试与匿名评测均可在仓库内复现。

## 10. 设计取舍

选择“预览优先”而不是“对话优先”，是因为该任务的核心风险是结构漏项和错误写入。对话适合解释与局部修正，但不应成为核对几十个任务的唯一界面。

选择“Code 确定性生成 + AI 语义质检”，是因为节点覆盖、字段合法性和幂等创建需要可重复、可测试；AI 更适合判断命名、分组语义和异常解释。
